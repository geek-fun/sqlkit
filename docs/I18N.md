# Multi-Language Support Implementation

This document describes the internationalization (i18n) implementation in SQLKit, which supports both English and Chinese (Simplified) languages.

## Overview

The multi-language support is implemented using [vue-i18n](https://vue-i18n.intlify.dev/), following the same pattern as the [dockit project](https://github.com/geek-fun/dockit).

## Structure

### Language Files

All language files are located in `src/lang/`:

- **`index.ts`**: Main i18n configuration and initialization
- **`enUS.ts`**: English (US) translations
- **`zhCN.ts`**: Chinese Simplified translations

### Translation Keys Organization

Translations are organized hierarchically:

```ts
const _translations = {
  common: {
    buttons: { save: 'Save', cancel: 'Cancel' },
    status: { connected: 'Connected', disconnected: 'Disconnected' }
  },
  pages: {
    connections: { title: 'Connections', subtitle: 'Manage your database connections' },
    settings: { title: 'Settings' },
    history: { title: 'History' }
  },
  components: {
    serverForm: { labels: { host: 'Host', port: 'Port' } },
    serverCard: { status: { active: 'Active' } }
  }
}
```

## Language Detection and Storage

The language preference is:

1. **Stored in localStorage** with key `'lang'`
2. **Auto-detection**: If set to `'auto'`, follows system language (`navigator.language`)
   - If system language is `'zh-CN'`, uses Chinese
   - Otherwise, defaults to English

## Usage in Components

### In Vue Components

Import and use the `useI18n` composable:

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t } = useI18n()
</script>

<template>
  <h1>{{ t('pages.connections.title') }}</h1>
  <p>{{ t('pages.connections.subtitle') }}</p>
</template>
```

### With Interpolation

For dynamic content, use interpolation:

```vue
<template>
  <!-- English: "No connections match your search \"test\"" -->
  <!-- Chinese: "没有匹配\"test\"的连接" -->
  <p>{{ t('pages.connections.empty.message', { query: searchQuery }) }}</p>
</template>
```

## Language Switcher

Users can change the language in **Settings** page under **Appearance** section. The language selector offers three options:

1. **Auto (Follow System)** - `'auto'`
2. **English** - `'enUS'`
3. **简体中文** - `'zhCN'`

When changed, the preference is saved to `localStorage` and the UI updates immediately.

## Adding New Translations

To add translations for a new feature:

1. **Add keys to both `enUS.ts` and `zhCN.ts`**:

```typescript
// In enUS.ts
export const enUS = {
  // ... existing translations
  pages: {
    myNewPage: {
      title: 'My New Page',
      description: 'This is a new page',
    }
  }
}

// In zhCN.ts
export const zhCN = {
  // ... existing translations
  pages: {
    myNewPage: {
      title: '我的新页面',
      description: '这是一个新页面',
    }
  }
}
```

2. **Use in component**:

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t } = useI18n()
</script>

<template>
  <h1>{{ t('pages.myNewPage.title') }}</h1>
  <p>{{ t('pages.myNewPage.description') }}</p>
</template>
```

## Testing

To test translations:

1. Run the application: `npm run tauri dev`
2. Navigate to **Settings** page
3. Change the **Language** dropdown to test different languages
4. Navigate through different pages to verify translations
5. Try creating/editing connections to test form translations

## Implementation Reference

This implementation follows the pattern used in the [dockit project](https://github.com/geek-fun/dockit/blob/master/src/lang/index.ts), using the same `vue-i18n` library and similar configuration.
